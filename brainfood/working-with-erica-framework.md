## COGNITIVE ARCHITECTURE

### Visual-Spatial Memory (Mind Palace)
- Information is stored spatially with visual anchors
- She will SEE where she wrote something (notebook color, page location, ink color)
- Organization enables retrieval; chaos blocks access
- Color-coding and visual distinction are functional requirements, not aesthetic preferences

**When working with her:**
- Make things visually distinct (colors, gradients, emojis, styling)
- Support physical writing in notebooks (creates spatial memory anchors)
- Provide organizational structure before content
- Use visual formatting to help information stand out

### Bridge-Building Between Concepts
- Questions aren't revealing gaps - they're constructing cognitive bridges
- "Is that like when my CMS did X?" means she's finding the connection point
- She has extensive existing knowledge; she needs translation, not teaching from scratch
- Pattern recognition operates at high speed once bridges are built

**When working with her:**
- Build bridges to existing knowledge (EverQuest CMS, XAMPP, client-server architecture)
- Don't explain fundamentals - translate vocabulary between old and new frameworks
- Support bridge-building explicitly: "Yes, that's exactly like when..."
- Watch for questions as architecture formation, not confusion

### Structure Before Exploration
- Foundation first, then experimentation
- She needs the map before the journey
- Without framework, exploration becomes chaos she can't integrate
- Reverse-engineers from end result to understand structure

**When working with her:**
- Always provide Session 0 glossary/framework before new projects
- Show end result/example first, let her reverse-engineer the path
- Don't skip to exciting parts; build reference framework first
- Make glossaries with REAL vs ARBITRARY distinctions (she comes from languages with dictionaries)
